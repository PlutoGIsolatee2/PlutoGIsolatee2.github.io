🍞 SugarCube 存档编辑器 v0.2603.5

一款专门编辑 SugarCube 游戏存档的树形工具 · 支持 Base64、JSON 树、黑白名单、实时搜索、历史撤销/重做

🚀 快速上手

1. 在顶部 Base64 区域 粘贴或选择存档文件（.txt），点击 解 按钮解码。
2. 编辑左侧树节点：点击节点键名可选中，展开/折叠容器。
3. 使用下方 简单编辑面板 快速插入/删除节点，或使用 全量编辑面板 (CodeMirror) 精细修改当前节点。
4. 修改完成后，点击 编 按钮编码并复制到剪贴板，或下载为 .txt 文件。

💡 所有操作都会自动保存到浏览器 IndexedDB，刷新页面不会丢失进度。

📦 核心概念

  • 存档结构：SugarCube 存档本质是 JSON 对象，编辑器将其展示为可展开的树。根节点固定为 root（面包屑显示为 $）。
  • 节点类型：每个值旁会显示类型标签（string, number, boolean, null, object, array）。
  • 当前节点：高亮背景的行表示当前选中节点，所有编辑操作（新建、删除、全量编辑）都基于此节点。
  • 历史记录：树结构的每一次修改（新建/删除/推送）都会记录，可通过右下角悬浮按钮 ↩ / ↪ 撤销/重做。

📋 界面分区详解

🔐 Base64 功能区

  • 复制/粘贴：操作 Base64 文本。
  • 选择文件：读取本地的 .txt 存档文件，并自动解码。
  • 下载 txt：将当前 Base64 保存为 .txt 文件。
  • 解 / 编：悬浮在右下角的两个按钮。解码后树视图更新；编码后自动复制到剪贴板，并更新 Base64 文本框。
⚠️ 解码/编码前若全量编辑器中有未推送的更改，会弹出确认框，避免数据丢失。

🌳 树视图 & 面包屑

  • 点击 ▶/▼ 展开/折叠节点。容器节点显示 Object 或 Array(n)。
  • 点击 键名 或 值文本 可将其设为当前节点（高亮）。
  • 面包屑导航显示当前路径：$ / root / state / variables，点击任意段可跳转。
  • 叶节点值直接以文本展示（不可编辑，需通过全量面板修改）。

➕ 简单编辑面板

位于树下方，包含：

  • ➕ 新建节点：将下方 JSON 文本插入到当前节点下。
  • ✖ 删除当前节点：删除选中的节点（根节点不可删）。
  • ? 按钮：鼠标悬停可查看新建规则：
    – 当前节点为数组：输入数组会合并到末尾，其他类型追加到末尾。
    – 当前节点为对象：只有输入对象才会合并，其他类型无效。
  • 文本区内置 CodeMirror，支持语法高亮、括号匹配。

✎ 全量编辑面板

  • ✎ 编辑当前节点：将当前节点 JSON 格式化后载入编辑器（若节点超大（>2MB）会弹出确认）。
  • ⏎ 推送修改：将编辑器内容解析后更新到树视图，并清空编辑器。
  • 🗑️ 清空编辑器：清空当前编辑内容（若未推送会提醒）。
  • ↩ / ↪：编辑器内部的撤销/重做（与树的历史独立）。
  • ✨ 格式化：美化 JSON 格式（重新解析后序列化）。

编辑器支持语法高亮、自动补全、实时错误检查、代码折叠、搜索替换（Ctrl+F）等。

🔍 搜索与黑白名单

🔎 搜索

  • 两种模式：点击 $ 按钮切换文本搜索或 JSONPath 搜索。按钮背景色深表示 JSONPath 模式。
  • 文本搜索：在键名和值中（字符串形式）不区分大小写查找。受黑白名单影响。
  • JSONPath 搜索：使用 JSONPath 语法（如 $.root.state、$..price）精确查询，不受黑白名单限制（已在帮助中说明）。
  • 输入时自动实时搜索（防抖 300ms），也可点击“搜索”按钮。匹配数显示在右侧，可用 ◀/▶ 跳转。
  • 搜索到的节点会自动展开所有祖先并高亮，同时滚动到视野中央。
$.root.state.variables.gold
$..author

⚪⚫ 黑白名单

  • 点击 ⚪ 白名单 将当前节点加入白名单；⚫ 黑名单 加入黑名单。
  • 通过 🔘 未启用 按钮可循环切换过滤模式：白名单/黑名单/无。
  • 启用白名单后，只有位于白名单节点及其子树内的节点会被搜索到；黑名单则排除该节点及其子树。
  • 🗑️ 清空当前启用名单：根据当前模式清空白名单或黑名单（无模式时清空全部）。
  • 添加/清除名单后，若存在搜索词会自动重新搜索。

📌 黑白名单影响的是普通文本搜索，JSONPath 搜索不受影响。

🔄 历史与持久化

⏳ 撤销/重做

  • 树结构的每次修改（新建、删除、推送、解码）都会记录历史，最多 50 步。
  • 使用右下角悬浮按钮 ↩ / ↪ 进行全局撤销/重做。
  • 全量编辑器内部也有独立的撤销/重做按钮，仅影响编辑器内容。

💾 IndexedDB 自动保存

  • 当前树结构、展开状态、黑白名单、历史记录、编辑器内容等都会自动存入浏览器 IndexedDB。
  • 刷新页面或重新打开编辑器时，自动恢复上次工作状态（搜索状态除外，保持干净）。
  • 页面关闭或隐藏时也会尝试保存，最大限度防止数据丢失。
📎 小贴士
  • 如果编辑了节点但忘记推送，切换节点或执行解码等操作时会弹出提醒，防止丢失。
  • 大节点（>2MB）加载到全量编辑器前会弹出确认，避免浏览器卡顿。
  • 鼠标悬停在任何 ? 按钮上均可查看上下文帮助（新建节点规则、搜索说明）。
  • 悬浮按钮组（解、编、撤销、重做）固定在右下角，始终可及。
  • 所有操作均配有 aria-label，屏幕阅读器友好。
